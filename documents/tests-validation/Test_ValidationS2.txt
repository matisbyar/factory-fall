Les tests de validation pour les user story sprint 2 :



User story :
Interface graphique du jeu

Descente automatique des pièces

Gestion du score

Rotation des pièces.

Gestion du rang






Test pour interface graphique du jeu :

Nous avons une vue jeu qui montre le jeu avec les informations du score, rank, nom joueur.

Nous avons une vue de début de partie qui comprend.
Le nom du jeu, bouton démarrer, ** bouton joueurs (personnalisé), bouton commandes , bouton Exit ** non ajouté

Nous avons une vue game over qui affiche une vue si la partie est terminée.
il y a 2 boutons un pour Exit et un pour rejouer.




Test pour descente automatique des pièces :
 La pièce descend toute seule au bout du temps → timer attribué.
 Peut être modifié pour ajouter de la difficulté si besoin.





Test pour gestion du score :

Le score est dépendant à un joueur.

Le score s'incrémente automatiquement si le joueur complète des lignes.

Mise en relation en fonction du rank.







Test pour rotation des pièces :

Possibilité de tourner une pièce à 90° vers la gauche indéfiniment.

Possibilité de tourner une pièce à 90° vers la droite indéfiniment.

Vérification qu' il n'y aura pas de problème de collision en cas de rotation.





Test pour gestion du rang :

Le rank est en relation avec le score.

Le rank s'incrémente automatiquement quand le score augmente selon un palier.
